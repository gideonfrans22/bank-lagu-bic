# AGENTS.md

## Cursor Cloud specific instructions

### Project overview

Bank Lagu is a React SPA (Vite + TypeScript) with Supabase as the backend (Postgres + Auth + RLS). There is no custom backend server; the frontend talks directly to Supabase APIs.

### Running locally

Standard commands are in `package.json`:
- `npm run dev` — Vite dev server (port 5173)
- `npm run build` — TypeScript check + production build
- `npm run lint` — ESLint

### Supabase local emulator

The app requires a Supabase instance. For local development, use the Supabase CLI emulator:

1. Docker must be running (`sudo dockerd` if needed; ensure socket is accessible with `sudo chmod 666 /var/run/docker.sock`).
2. `npx supabase start` — starts Postgres, Auth, Storage, etc. in Docker containers.
3. `npx supabase db reset` — applies all migrations from `supabase/migrations/`.
4. Get credentials: `npx supabase status -o env` outputs `ANON_KEY` and `API_URL`.

The `.env` file (not committed) must contain:
```
VITE_SUPABASE_URL=http://127.0.0.1:54321
VITE_SUPABASE_ANON_KEY=<anon key from supabase status>
```

After changing `.env`, the Vite dev server must be restarted for changes to take effect.

### Gotchas

- There is no automated test suite (no Jest, Vitest, or Playwright configured). Validation is via lint, type-check (`tsc -b`), and manual testing.
- The `supabase/config.toml` is generated by `npx supabase init --force` if missing; it is not committed to git.
- The Docker nested-container setup requires `fuse-overlayfs` storage driver and `iptables-legacy`. See the environment setup for details.
- Supabase local emulator auth does NOT send real emails — use the Mailpit UI at `http://127.0.0.1:54324` to receive confirmation emails if email confirmation is enabled.
- The `supabase/.temp/` directory contains linked project metadata from a remote Supabase project; it can be ignored for local development.
